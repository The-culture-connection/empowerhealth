Fix ADMIN Document Structure

This will migrate your ADMIN document from auto-generated ID to uid-based ID.

Your current situation:
Document ID: ASftE2NiYG8sQzQfd48x (auto-generated)
Should be: pPw9YOT2DGWLDqREGPPRAef6zh82 (your uid)

Option 1: Quick Fix via Browser Console

Open your browser console (F12) on the admin dashboard and run:

// Get your current user
const auth = getAuth();
const user = auth.currentUser;

if (!user) {
  console.error('Not logged in');
} else {
  const db = getFirestore();
  const adminRef = collection(db, 'ADMIN');
  const q = query(adminRef, where('email', '==', user.email));
  
  getDocs(q).then(snapshot => {
    snapshot.forEach(async (docSnap) => {
      const data = docSnap.data();
      if (docSnap.id !== user.uid) {
        console.log('Migrating:', docSnap.id, '→', user.uid);
        // Create new document with uid as ID
        await setDoc(doc(db, 'ADMIN', user.uid), data);
        // Delete old document
        await deleteDoc(doc(db, 'ADMIN', docSnap.id));
        console.log('✓ Migration complete! Refresh the page.');
      }
    });
  });
}
